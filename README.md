# AI事前問診メモ - Phase 0 プロトタイプ

患者が音声で症状を話すと、AIがメモに整理して、LINEなどで送信できるPWA（Progressive Web App）です。

## 📦 ファイル構成

```
ai-prediagnosis-pwa/
├── index.html          メインアプリ（UI）
├── app.js              アプリケーションロジック
├── manifest.json       PWA設定
├── service-worker.js   オフライン対応
├── icons/
│   ├── icon-192.png    アプリアイコン (192x192)
│   └── icon-512.png    アプリアイコン (512x512)
├── start.sh            ローカル起動スクリプト
└── README.md           このファイル
```

## 🚀 起動方法

### 方法1: 起動スクリプトを使う

```bash
cd ai-prediagnosis-pwa
./start.sh
```

### 方法2: Python のHTTPサーバーを直接起動

```bash
cd ai-prediagnosis-pwa
python3 -m http.server 8000
```

その後、ブラウザで以下を開く：

- **PCのブラウザで確認:** http://localhost:8000

### 方法3: スマートフォンで確認する

1. PCとスマホを **同じWi-Fi** に接続
2. PCのIPアドレスを調べる
   - Mac: `ifconfig | grep "inet " | grep -v 127.0.0.1`
   - Windows: `ipconfig`
3. スマホのブラウザで `http://<PCのIP>:8000` にアクセス
   - 例: `http://192.168.1.100:8000`

> ⚠️ **マイク使用について**
> ブラウザのセキュリティ仕様により、マイク機能は以下の環境でのみ動作します：
> - `http://localhost` または `https://...`
> - そのため、スマホで試す場合は **HTTPS化が必要** です（後述の ngrok を参照）

## 📱 スマホで試す（HTTPS必要時）

### ngrok を使う方法

```bash
# 1. ngrokをインストール（https://ngrok.com/）
# 2. ローカルサーバーを起動
python3 -m http.server 8000

# 3. 別ターミナルで ngrok を起動
ngrok http 8000
```

ngrok が発行するHTTPS URL（例: `https://xxxxx.ngrok.io`）をスマホで開く。

## 🔑 OpenAI APIキーを設定する（オプション）

**APIキーなしでもモック動作で全機能を試せます**。実際のAIで動かしたい場合：

1. https://platform.openai.com/ でAPIキーを発行
2. アプリの設定画面（右上の⚙️）を開く
3. 「OpenAI APIキー」欄に `sk-...` を貼り付け
4. 「保存」ボタンをタップ

→ 以降、Whisper（音声認識）とGPT-4o（要約）が実際に動作します。

> ⚠️ **セキュリティ警告**
> このプロトタイプはAPIキーをブラウザのlocalStorageに保存します。
> 本番運用ではAPIキーをサーバー側で管理する必要があります。

## 🎯 機能一覧

### ✅ 実装済み（Phase 0）

- 🎙️ **音声録音**: マイクボタンで録音/停止
- 🤖 **AI要約**: 音声 → 文字起こし → 構造化メモ
  - 本物のOpenAI Whisper + GPT-4o対応
  - APIキー未設定時はモック動作（5パターン）
- 📝 **主訴確認**: AIが認識した主訴を患者が確認・修正
- ✏️ **メモ編集**: 各項目をタップして編集可能
- 📤 **LINE等で送信**: Web Share APIでOS標準シェア
  - 非対応ブラウザではクリップボードコピー
- 💾 **履歴保存**: 過去のメモをlocalStorageに保存
- ⚙️ **設定画面**: APIキー管理、履歴管理

### 🚧 Phase 1（B-2 LIFF版）で実装予定

- LINE Front-end Framework (LIFF)対応
- クリニックLINE公式アカウントへの直接送信
- LINE IDによる本人特定
- リッチメニューからの起動

## 🎨 デザインのポイント

- **高齢者にやさしいUI**: 大きな文字（17px〜）、大きなボタン
- **清潔感ある医療系カラー**: ブルー基調 + グリーンアクセント
- **わかりやすい画面遷移**: 1画面1メッセージの原則
- **録音中の視覚フィードバック**: 音波アニメーション、タイマー
- **編集の容易さ**: タップで項目編集

## 🏗️ 技術スタック

- **フロントエンド**: 純粋なHTML/CSS/JavaScript（フレームワーク不使用）
- **音声録音**: MediaRecorder API
- **AI連携**: OpenAI Whisper API + GPT-4o
- **PWA**: Service Worker + Web App Manifest
- **シェア**: Web Share API（フォールバック: Clipboard API）

## 📝 注意事項

このプロトタイプは **患者本人が自分用のメモを作成する** ためのアプリです。

- ❌ 医療機関の診療情報システムではありません
- ❌ AIによる診断・治療提案は行いません
- ✅ クリニックでのヒアリングは必ず別途実施されます
- ✅ メモの送信は患者本人の意思で行います
