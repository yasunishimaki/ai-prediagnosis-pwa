# デモ手順書（7月2日 クレアスクリニック）— Cloudflare Pages 版

患者役（スマホ）と受付役（iPad）の2台を使う一連デモの手順。
配信は **Cloudflare Pages**。OpenAIキーは**サーバー側（Cloudflare）に設定済み**なので、端末ごとのキー入力は不要。

- 患者用: `https://qureas-monshin.pages.dev/`
- 受付用: `https://qureas-monshin.pages.dev/reception-app/`

---

## 0. 事前準備（前日まで）

- [ ] **患者用スマホ**でURLを開けるか確認 → `https://qureas-monshin.pages.dev/`
- [ ] 画面右上が **🟢 本番API（サーバー経由）** になっているか（＝キー設定済みで動く状態）
- [ ] **受付用iPad（Safari）**でURLを開けるか確認 → `https://qureas-monshin.pages.dev/reception-app/`
- [ ] スマホの**マイク許可**、iPadの**カメラ許可**を一度通しておく
- [ ] ⚠️ **スマホのBluetoothをオフ**にする（イヤホン等が繋がっていると本体マイクが無音になり、文字起こしが壊れることがある）
- [ ] iPadと**AirPrint対応プリンタ**を同じWi-Fiに接続し、テスト印刷
- [ ] 通信が不安な会場では、スマホのテザリング等の代替回線を用意
- [ ] OpenAI側の**使用上限（予算アラート）**が設定されているか確認

> 💡 ネットが完全に不通でも、QR/読取/印刷はオフラインで動作（音声認識のみ要ネット）。
> 通しの流れだけ見せたい場合は、別端末の開発用モードで固定シナリオを使う手もある。

---

## 1. 患者デモ（スマホ）

1. ホームの「**🎙️ 問診を始める**」をタップ
2. マイクボタンを押し、症状を自由に話す
   - 例: 「3か月くらい前から頭がズキズキ痛くて、夕方に強くなります。市販の頭痛薬を飲んでいます」
3. もう一度マイクを押して録音停止 → AIが分析
4. **AIが不足項目を1つずつ質問**するので、マイクを押して音声で回答
   - 例: 「アレルギーはありますか？」→「特にありません」
   - 「測っていないのでわからない」等は、そのまま言えば**「不明」で確定**して次へ進む
   - 答えにくい質問は「**この質問はわからない / 答えない**」でスキップ可
5. 全項目が終わると**問診メモ**が表示される。項目をタップで修正可
6. 「**✅ この内容で受付に提示する（QR表示）**」をタップ → QRコード表示

> 💡 聞き取れなかったときは、オレンジの「**🔄 うまく聞き取れませんでした。もう一度おうかがいします**」バーが出て同じ質問を再表示する。**エラーではなく聞き直し**なので、落ち着いてもう一度話せばOK。

---

## 2. 受付デモ（iPad）

1. 受付アプリで「**QRコードを読み取る**」をタップ
2. 患者スマホのQRをカメラ枠に合わせる → 自動で読み取り
3. **問診メモが表形式で表示**される（服薬・アレルギーは黄色で強調）
4. 「**🖨️ 印刷する**」→ AirPrintでプリンタを選んで印刷
5. 「**📷 次の患者さん**」で続けて読み取り可能

---

## 3. 説明トーク（ポイント）

- クリニックごとに**聞き取る項目をテンプレートで設定**できる（今回はクレアス＝一般内科）
- AIが**不足だけを補う**ので、患者の負担が少なく、受付で要点が揃う
- 文字起こし・要約はサーバー側（Cloudflare Pages Functions）経由で、**OpenAIキーは患者端末に出ない**
- 問診メモ自体は**QRで渡してその場完結＝サーバーに患者データを保存しない**（プライバシー配慮の設計）

---

## 4. トラブル時の対処

| 症状 | 対処 |
| --- | --- |
| 文字起こしが壊れる／変な文（「チャンネル登録を…」等） | **Bluetoothを切る**（本体マイクが無音になっている）。静かな場所で話す |
| マイクが反応しない | URLが `https` か確認。ブラウザのマイク許可をオン。ページ再読込 |
| 同じ質問が出た（エラーに見える） | オレンジの聞き直しバーが出ていれば**正常な聞き直し**。もう一度話す |
| カメラが起動しない（iPad） | Safariのカメラ許可を確認。他アプリがカメラ使用中なら閉じる |
| QRが読み取れない | 画面の明るさを上げ、QRを枠いっぱいに。距離20〜30cm |
| QR生成に失敗 | メモの文字数が多すぎる場合あり。長い項目を短く編集して再表示 |
| 「Server is not configured」表示 | Cloudflare の環境変数 `OPENAI_API_KEY`（シークレット）と再デプロイを確認 |
| AIの追加質問が長い／止まらない | 「わからない/答えない」でスキップして次へ |
| 音声認識がうまくいかない | Bluetoothオフ・静かな場所で。最悪はメモ画面で項目を手入力修正 |
